# 首批公共基础组件说明

## 范围

当前首批公共基础组件统一收敛在 `components/ui/`，目标是为业务页面提供稳定依赖，同时为未来真实 `uni-ui x` 导入预留替换点。

## 组件清单

### `UiNavBar`

- 用途：承载页面标题、副标题及右侧扩展内容
- 当前形态：仓库内占位桥接实现
- 后续方向：优先映射到 `uni-nav-bar`

### `UiSectionCard`

- 用途：承载页面中的分组卡片、说明块和轻量内容容器
- 当前形态：项目自研基础容器
- 后续方向：默认保持自研，不强制映射到 `uni-ui x`

### `UiEmptyState`

- 用途：承载空态、未接入提示和准备态说明
- 当前形态：项目自研空态组件，内置“占位桥接”标识
- 后续方向：按业务需要组合真实组件能力，不强制直接替换

## 使用约束

- 业务页面只依赖 `components/ui/*`
- 不在业务代码里直接引用未来 `uni_modules/*` 真路径
- 真实组件导入后，优先改桥接层，不直接改页面层
- 当前仓库不包含任何伪造的 `uni-ui x` 第三方源码
